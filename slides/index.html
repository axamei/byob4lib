  • On the left is a condition known as divitis. Prone to happen in Bootstrap. Excessive use of nested divs.
  • div class=”col-md-6” is not semantic. Instead of using all those col-spans, use selectors that convey the context of your content.
  • On the right BYOB version. Less presentational markup and css classes. Less nested divs. Contextual, semantic sections. Less messy. More maintainable.

Designers vs. Bootstrap

  • Great for web development, horrible for web designers... It's the times new roman of the web. All sites basically look the same. Too prescriptive. The canvas is not blank.
  • Bootstrap is a development solution. Not a design one.
  • Designers want more flexibility and creative freedom
  • Coupling of HTML and CSS leads to col- and div- sanity and lack of creative freedom
  • Yes, it's somewhat customizable, but why bother?
  • If you are already investing heavily in customizing bootstrap, you might want to just make pull your own together.
  • Put your coding and design skills to better use.

Let's talk about Snowflakes

  • First Problem: Usability Study: Why do all your sites look totally different?
  • Problem that libraries face that we want a framework like Bootstrap to help us solve: consistency! better design! Teams with no designers: gets-the-job-done design that doesn’t suck but is also very, very generic.

No shortage of platforms

  • Second Problem: All of these websites run on different platforms.
  • Open: ILLiad/Wordpress/Rails Applications/In House
  • Vendors with Source code access Markup/CSS: ILLiad/Ex Libris
  • Vendors who let you decorate: Springshare (libguides), Summon, "header/footer" embeds.
  • When we began to seriously evaluate things layering a prescriptive framework like Bootstrap on all these existing systems requires a lot of fighting and alteration of existing markup and additional CSS clases.
  • We certainly *could* implement bootstrap or parts of it on all these projects but we would be tightly coupling our work to it's receipes and conventations.
  • We wanted to come up with a solution that put us on a scalable path to maintaining a wide variety of sites in a variety of platforms.

  • main scss file and what our modular dir structure looks like. Intially inspired by Drupal's Omega 4 Theme.
  • Abstractions contains Sass mixins, extends and functions for use throughout your stylesheets and help to promote code REUSE.
  • Base styles define the default look for HTML and, to some extent, common elements. e.g. headings
  • Components: Components are discrete parts / blocks of your page that should sit within the regions of your layouts. You should try to abstract your components as much as possible to promote reuse throughout the theme.
  • Also, let’s you grab what you want, grab what you need, or grab the lot; the choice is yours.

Reusable Code

SCSS:

$error-color: #fbe3e4;
$success-color: #e6efc2;

@mixin notification($color) {
  background-color: $color;
  color: darken($color, 60%);
  display: block;
  font-weight: 600;
  margin-bottom: 0.75em;
  padding: 0.75em;
  text-align: center;
}

.notification__error {
  @include notification($error-color);
}

.notification__success {
  @include notification($success-color);
}

Patterns, Components & Type Styles, Oh My!

  • Refills copy and paste HTML/Scss snippets of patterns, components, type styles
  • It's like Bootstraps pattern library, but you can and are encouraged to customize these components. Use your own selectors. Keep your HTML and CSS decoupled
  • Refills default to using the built-in to Neat grid system so the components are packaged to be responsive. It's not so plug n play, but they are easily adjustable to fit other abstract grid systems. (Insert Demo)
  • This stack of Sass libraries like Singuarity and Breakpoint gives us the "Killer" Bootstrap Features and alternative solutions / tools to allow creative freedom, de-coupling of markup from presentation, and support rapid iteration.

// Twitter Bootstrap grid
@include add-grid(12);
@include add-gutter(1/3);





// Custom asymmetrical grid
@include add-grid(1 4 4);
@include add-gutter(1/6);
@include add-gutter-style(split);




// Ratio spiral function 
@include add-grid(ratio-spiral(5));
@include add-gutter(1);
  • gem, require, import
  • Ratio based, Semantic Grid System. This means that instead of grid classes being generated that you apply to your HTML, the grid is stored entirely within your CSS and applied directly to the element you want to use it on.
  • It’s easy and you can make your own! Define grid, gutter and gutter style

Breakpoint

SCSS:

// Breakpoints
$medium: 46em;
$large: 72em;

.main {
  @include breakpoint($medium) {
    content: 'Keanu approved';
  }
}

CSS:

@media (min-width: 46em) {
  .main {
    content: 'Keanu approved';
  }
}
Whoa.
  • Bootstrap has 4 breakpoints. xs, sm, md, lg.
  • What about xl or xxl? Or if you want fewer breakpoints
  • RWD should work with the content itself, not impose a class structure.
  • Breakpoint gem is so easy Keanu approves

Responsive Grids

Grid level:

@include add-grid(.25 8 .25);
@include add-grid(2 3 at $medium);
@include add-grid(1 4 1 at $large);

@include add-gutter(.25);
@include add-gutter(1/6 at $medium);
@include add-gutter(1/3 at $large);

Block level:

.main {
    background: red;
    @include grid-span(3, 1);

    @include breakpoint($medium) {
        @include grid-span(1, 2);
    }
    @include breakpoint($large) {
        @include grid-span(1, 2);
    }
}
